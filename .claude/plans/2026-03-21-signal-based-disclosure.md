# Plan: Signal-Based Disclosure Architecture (The Iceberg UI)

## Objective
Refactor the audit results and main interface to prioritize visual signals and iconography. Dense copy and technical language will be "submerged" behind tap/click interactions, ensuring the primary experience is visual and "at-a-glance" (The Iceberg Tip).

## Implementation Steps

### 1. Visual Score Pillars (The Gateway)
- **Change:** Instead of a simple chart or list, create a "Pillar Grid" on the Results page.
- **Visual:** Each of the 4 cosmic pillars (Mercury, Venus, Mars, Saturn) will be represented by a high-fidelity icon/glyph and a large "Alignment Score."
- **Interaction:** Tapping a pillar will "reveal" the submerged text (Overview, Bottlenecks, etc.) using a slide-up or fade-in transition.

### 2. Signal-Based Action Cards
- **Change:** Replace the dense `UpsellCard` and "Next Steps" text with visual "Path Nodes."
- **Visual:** Minimal icons (e.g., a Hammer for The Builder, a Key for The Vault, an Eye for The Oracle).
- **Interaction:** Tapping a node expands the description and CTA.

### 3. Submerged Markdown (The Ocean)
- **Change:** The full Markdown audit generated by the AI will be initially hidden.
- **Visual:** A "Deep Intelligence" toggle or "Explore Submerged Report" button at the bottom of the signal grid.
- **Interaction:** Clicking this will reveal the full technical report for users who want to dive deep.

### 4. Global CSS: Signal Utility
- **Standard:** Implement `--signal-opacity` and `--tap-feedback` variables.
- **Animation:** Use `view-transition` or CSS `height` transitions to make the "disclosure" feel like a descent into the deep.

## Key Files
- `src/app/results/page.tsx`: Re-architect layout to be signal-primary.
- `src/components/CosmicChart.tsx`: Enhance or replace with an interactive glyph-based pillar grid.
- `src/components/UpsellCard.tsx`: Refactor for minimal visual noise.
- `src/app/globals.css`: Add motion primitives for progressive disclosure.

## Verification
- **Mobile Check:** Ensure the "Signals" are large and easily tappable (min 44px).
- **Visual Audit:** Confirm that 90% of the screen real estate is initially visual/signal-based.
- **Manual Flow:** Verify that all dense information is accessible via a secondary tap.
